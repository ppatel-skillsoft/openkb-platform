# Implementation Plan: FastMCP Knowledge Base Server

**Branch**: `feature/009-fastmcp-kb-server` | **Date**: 2026-06-26 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/009-fastmcp-kb-server/spec.md`

## Summary

Add a **FastMCP 3.4.2 HTTP server** (`mcp_server/`) that exposes two read-only MCP tools ‚Äî `ask_kb` and `list_kbs` ‚Äî to any MCP host (Claude Desktop, Cursor, GitHub Copilot). The server proxies queries to the existing `generator-api` service and reads KB readiness from PostgreSQL. It runs as a new Docker Compose service on port 8002. A companion ingestion script (`scripts/ingest_marketing_kb.py`) uses exponential-backoff retry (tenacity) and an asyncio semaphore to submit the real-world marketing documents from `marketing_kb/` without triggering OpenAI 429 errors.

## FastMCP 3.x Design Patterns Applied

### 1. Lifespan for shared `httpx.AsyncClient`

```python
# mcp_server/app.py
from fastmcp.server.lifespan import lifespan

@lifespan
async def _app_lifespan(server):
    settings = get_settings()
    async with httpx.AsyncClient(
        base_url=settings.generator_api_url,
        timeout=httpx.Timeout(settings.query_timeout_s),
    ) as client:
        yield {"http_client": client}

mcp = FastMCP(
    "OpenKB",
    instructions=(
        "Query compiled knowledge bases. "
        "Use list_kbs to discover available knowledge bases, "
        "then ask_kb to get grounded answers with citations."
    ),
    lifespan=_app_lifespan,
)
```

### 2. Tools with typed Pydantic input, Context injection, and ToolAnnotations

```python
# mcp_server/tools/ask_kb.py
from fastmcp import Context
from fastmcp.tools import ToolAnnotations

@mcp.tool(
    annotations=ToolAnnotations(readOnlyHint=True, idempotentHint=True, openWorldHint=True),
)
async def ask_kb(kb_id: str, question: str, ctx: Context) -> KBAnswer:
    """Query a compiled knowledge base with a natural-language question.

    Args:
        kb_id: UUID of the knowledge base. Obtain from list_kbs.
        question: Natural-language question. Maximum 8000 characters.
    """
    _validate_kb_id(kb_id)
    _validate_question(question)
    client: httpx.AsyncClient = ctx.lifespan_context["http_client"]
    ...
```

### 3. Health endpoint via `@mcp.custom_route`

```python
# mcp_server/app.py
from starlette.requests import Request
from starlette.responses import JSONResponse

@mcp.custom_route("/health", methods=["GET"])
async def health(request: Request) -> JSONResponse:
    # lightweight ping to generator-api /health
    ...
```

### 4. ASGI deployment via `mcp.http_app()`

```python
# mcp_server/app.py
http_app = mcp.http_app(path="/mcp")
```

`Dockerfile.mcp-server` entry: `uvicorn mcp_server.app:http_app --host 0.0.0.0 --port 8002`

### 5. Auto-deref schemas (default `True`) for Claude Desktop / VS Code Copilot compatibility

FastMCP 3.x automatically dereferences `$ref` entries in tool schemas before sending to clients. No explicit configuration needed ‚Äî this ensures Pydantic models with shared types work with all MCP hosts, including those that do not fully support JSON Schema `$defs`.

## Ingestion Script Design

```mermaid
flowchart TD
    A[Scan marketing_kb/ recursively] --> B{Supported extension?}
    B -- No --> SK[Skip with warning]
    B -- Yes --> C[Check if KB exists\nPOST /kbs if not]
    C --> D[asyncio.Semaphore 3\nAcquire slot]
    D --> E[200ms pre-submission delay]
    E --> F[POST /kbs/{id}/documents\nwith file upload]
    F --> G{HTTP 429 or 5xx?}
    G -- Yes, attempt <= 5 --> H[tenacity exponential backoff\nbase=2s mult=2 max=60s jitter]
    H --> F
    G -- Yes, attempt > 5 --> FAIL[Mark failed\nlog error]
    G -- No --> SUB[Mark submitted\nlog success]
    SUB --> I[Release semaphore slot]
    FAIL --> I
    SK --> J[Next file]
    I --> J
    J --> K{More files?}
    K -- Yes --> D
    K -- No --> L[Print summary\nExit 0 if no failures]
```
